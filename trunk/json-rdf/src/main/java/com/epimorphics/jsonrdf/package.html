Provides encoding of RDF resources, models or DataSets into a developer-friendly JSON form and decoding of the generated format back into RDF. Round-tripping is not guaranteed in all cases but is is possible in most cases if properties are used with consistent ranges.

The typical usage is:

Encoder.get().encode(model, resourcesList, writer)

which uses a default set of encoding rules and no context information. Mapping information can be specified by means of a Context object:

Context context = new Context([baseURL], [ontologyModel]);
Encoder.get(context).encode(...);

The ontologyModel provides default label mappings for properties and can tag properties to be hidden or treated as multivalued. See help file.

The other Encoder.get() options allow specification of a set of encoding rules via an EncoderPlugin. This is just to ease experimentation during development and the option is likely to be removed once the rules settle down.
